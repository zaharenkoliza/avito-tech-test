# Avito Tech Test

## Описание проекта

Приложение "AI-ассистент для улучшения объявлений на Авито".  Личный кабинет продавца с интегрированным AI-ассистентом, который помогает улучшать описания объявлений.

Приложение позволяет:
- посмотреть список объявлений продавца;
- открыть карточку объявления;
- отредактировать объявление;
- запросить у AI улучшение описания, подсказку цены и ответ в чате;
- сохранить черновик формы и восстановить его после перезагрузки.

Проект состоит из двух частей:
- `client/` - frontend на React + Vite;
- `server/` - backend на Fastify + TypeScript.

Основные пользовательские сценарии:
- список объявлений `/ads`;
- просмотр объявления `/ads/:id`;
- редактирование объявления `/ads/:id/edit`;
- AI-функции: генерация описания, подсказка цены, чат;
- переключение светлой и темной темы.

## Запуск проекта

### 1. Через Docker Compose

Самый простой способ поднять весь проект:

```bash
docker compose up --build
```

После запуска будут доступны:
- клиент: `http://localhost:5173`
- API: `http://localhost:8080`
- Ollama: `http://localhost:11434`

Сервисы в `docker-compose.yml`:
- `client` - dev-сервер Vite;
- `server` - Fastify backend;
- `api` - прокси `8080 -> 8081`, чтобы у клиента был стабильный адрес API;
- `ollama` - runtime для локальной LLM;
- `ollama-init` - загрузка модели при старте.

По умолчанию в Docker используется:

```bash
OLLAMA_MODEL=llama3:latest
```

Если нужен более быстрый первый запуск, можно выбрать более легкую модель:

```bash
OLLAMA_MODEL=llama3.2:3b docker compose up --build
```

### 2. Локально без Docker

#### Требования

- Node.js 20+
- npm 10+
- установленный и запущенный Ollama

#### Настройка LLM

1. Установить Ollama.
2. Подтянуть модель:

```bash
ollama pull llama3.2:3b
```

3. Убедиться, что Ollama запущена:

```bash
ollama serve
```

По умолчанию сервер проекта обращается к:

```bash
http://localhost:11434
```

#### Backend

```bash
cd server
npm install
npm start
```

Сервер по умолчанию стартует на `http://localhost:8080`.

Переменные окружения сервера:

```bash
PORT=8080
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=llama3.2:3b
OLLAMA_TIMEOUT_MS=120000
OLLAMA_RETRIES=1
```

#### Frontend

```bash
cd client
npm install
npm run dev
```

Клиент будет доступен на `http://localhost:5173`.

Если API поднят не на стандартном адресе:

```bash
VITE_API_BASE_URL=http://localhost:8080
```

## Архитектура

Проект разделен на клиент и сервер.

### Клиент

- `app` - точка входа, роутер, store, провайдеры;
- `pages` - orchestration на уровне маршрутов;
- `widgets` - крупные UI-блоки страниц;
- `features` - пользовательские сценарии и фичи;
- `entities` - доменные сущности и бизнес-утилиты;
- `shared` - общая инфраструктура: API, storage, utils, config.

### Сервер

- `server.ts` - конфигурация Fastify и HTTP-роуты;
- `src/ai/*` - интеграция с Ollama, промпты и типы;
- `src/validation.ts` - валидация входных данных;
- `src/types.ts` - типы доменной модели;
- `server/data/items.json` - исходные данные объявлений.

Дополнительно архитектура клиента описана в `client/README.md`.

## Особенности реализации

- список объявлений поддерживает поиск, фильтры, сортировку, пагинацию и переключение layout;
- состояние списка синхронизируется с URL;
- черновик формы редактирования сохраняется в `localStorage`;
- AI-запросы и загрузка страниц поддерживают отмену через `AbortController`;
- есть визуальное сравнение исходного и улучшенного описания;
- сервер добавляет искусственную задержку, чтобы было видно loading-state;
- данные backend хранятся в памяти и не переживают перезапуск сервера.

## Описание принятых самостоятельных решений

1. ID на backend
- изначально endpoint `GET /items` не возвращал поля `id` и `createdAt`;
- это пришлось исправить на backend, потому что без `id` нельзя было корректно строить переходы на карточку и редактирование, а без `createdAt` невозможно было полноценно отображать и сортировать список объявлений;
- после этого переходы на детали и редактирование стали строиться по `item.id`, а список получил необходимые данные для отображения.

2. Решение по CORS
- при локальной интеграции frontend и backend CORS мешал отправке запросов;
- на backend были добавлены заголовки `Access-Control-Allow-Methods` и `Access-Control-Allow-Headers`, а также отдельная обработка `OPTIONS` с ответом `204`;
- после этого preflight-запросы перестали ломать взаимодействие клиента с API, и frontend смог стабильно работать через `VITE_API_BASE_URL`.

3. Кнопка возврата к списку товаров
- в исходном макете явной кнопки возврата к списку товаров не было;
- кнопка была добавлена отдельно как улучшение пользовательского сценария;
- при этом она оформлена в стиле существующего макета, чтобы не выбиваться из общего интерфейса.

4. Переключатель темы
- при реализации переключателя темы возникла дилемма с размещением, потому что в интерфейсе нет отдельного `header` или `footer`;
- поэтому было решено сделать кнопку фиксированной в углу экрана, чтобы она оставалась доступной на всех страницах;
- выбранная тема сохраняется в `localStorage`.

5. `ollama:latest` слишком долго грузилась
- для локального запуска зафиксирована более легкая модель `llama3.2:3b`;
- в Docker оставлена возможность переопределить `OLLAMA_MODEL`;
- дополнительно были уменьшены промпты, чтобы модель получала меньше контекста и возвращала меньше лишней информации, это помогло сделать ответы быстрее и стабильнее.
